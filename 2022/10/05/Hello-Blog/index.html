五.在搭建过程中出现的问题及解决方法

1.1 安装git

  • Git官网下载。下载完成之后，点击安装包，选择全部默认，一路Next,唯一一点就是记得修改自己的下载路径，当然不改也行，之后就安装完成。

  • 测试：打开cmd（记得在所在文件之下），输入git –version.

1.2 安装Node.js

  • Node官网下载。下载完成之后，点击安装包，选择全部默认，一路Next,唯一一点就是记得修改自己的下载路径，当然不改也行，之后就安装完成。

  • 测试：打开cmd（记得在所在文件之下），输入node -v.

1.3 GitHub

  • 注册Github,因为之前我已经注册好了Github,这步当时也就省略了

2.1 Hexo下载

  • 在你想建成博客目录的盘符创建一个文件夹作为博客的顶级目录

  • 打开cmd，进入创建文件夹，输入npm install hexo-cli -g 安装hexo

2.2 Hexo初始化

  • 安装完成后，输入hexo init blog进行初始化
  • 初始化成功后，文件夹下就会有下方形成的目录结构

2.3 启动博客

  • cmd进入到这里，然后输入hexo s -g启动，博客就搭建完成了

3.1 添加自己用户名及邮箱到git配置中

3.2 生产SSH密钥

以下指令一路回车即可，密码不填默认无密码

3.3 密钥添加到github

  • 打开文件夹，可以在C盘找到该文件，用VScode打开并复制内容

  • 打开自己github的主页，点击右上角头像，打开setting
  • 点击New SSH key,添加密钥

4 创建Github Pagea 仓库

Github主页右上角加号 –>New repository:

  • Repository name中输入 用户名.github.io
  • 勾选”Initialize this repository with a README”
  • Description选填
    这几步完了之后点击Create repository创建

5.1 出现的问题：

写完博客之后打开并不是blog，而是github仓库里的Readme文件

解决方法：

出现这问题，当时又重新创建Github Pagea 仓库，部署Hexo到Github Pages弄了好几遍之后出来依旧是那样，并且当时还在bing上查阅了很多资料，试了很多种办法，依旧不行。最后，找问题的着重点放在了branch上，当时我在_config.yml里设置的branch是master,最后将master改为了main。

之后打开正常了。
